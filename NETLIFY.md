# Netlify deployment (Fast Education)

This project is **Django backend + static SPA frontend**. Netlify serves only the **static frontend**.

## What gets deployed on Netlify

- **Publish directory:** `public/` (generated by build)
- **Build command:** `bash scripts/build_netlify.sh`
- **SPA routing:** `_redirects` sends `/*` → `/index.html` 200

## Deploy steps

1. Connect the repo to Netlify (GitHub/GitLab/Bitbucket).
2. Build command: `bash scripts/build_netlify.sh` (or leave default from `netlify.toml`).
3. Publish directory: `public`.
4. Deploy. The site will load, but **API calls (`/api/*`) will fail** unless you proxy them.

## Backend (Django) required

The frontend calls `/api/register/`, `/api/levels/`, etc. You must run the Django backend elsewhere (e.g. Railway, Render, Fly.io, or your server) and either:

- **Option A:** Set the frontend API base URL to your backend (e.g. `https://your-api.railway.app/api`) and deploy Django there.
- **Option B:** In Netlify **Redirects**, add:  
  `/api/*` → `https://YOUR-DJANGO-URL/api/:splat` with status **200** (proxy).

## Local build check

```bash
bash scripts/build_netlify.sh
# Then open public/index.html or serve public/ with any static server
```

## Folder layout after build

```
public/
  index.html      # Static HTML (no Django tags)
  _redirects      # /* -> /index.html 200
  css/style.css
  js/app.js
  images/logo.png
  images/levels/*.jpg
```
